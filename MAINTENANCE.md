# 保守システム（あと稼げるくん）

このサイトは GitHub Actions で**自動保守**されています。あなたのPCやClaudeが起動していなくても、GitHubのサーバー上で定期実行されます（パブリックリポジトリのため無料）。

## 自動で動く3つの仕組み

| 仕組み | いつ動く | 何をする | 失敗したら |
|---|---|---|---|
| **テスト** (`ci.yml`) | push時・毎週月曜 | 住民税/データ検証の44テストを実行 | チェックが赤くなり、オーナーにメール |
| **死活・リンク監視** (`healthcheck.yml`) | 毎週火曜 | 公開サイトの稼働・内部リンク切れ・sitemap整合を確認 | 同上（メール通知） |
| **税制レビュー通知** (`tax-review.yml`) | 毎年12月・1月 | 「数値を見直して」というIssueを自動発行 | — |

GitHub の **Actions** タブで実行履歴を確認できます。

## 手元での確認コマンド

```bash
npm test               # 計算ロジックのテスト（44件）
npm run healthcheck    # 内部リンク・sitemapの整合チェック
npm run healthcheck:live  # 公開サイトの死活も含めてチェック
npm run check          # テスト＋健全性をまとめて実行
```

> 編集して push する前に `npm run check` が通ることを確認してください。

## 税制が変わったときの更新手順

毎年12月に自動発行される **税制レビューIssue** のチェックリストに従って、
`calc.js` の定数・`PREF_KINTOU_WARI` と各HTMLの数値を更新します。
詳細は [.github/tax-review-checklist.md](.github/tax-review-checklist.md) を参照。

更新後は必ず `npm run check` → push（テストが期待値と一致するよう `test/calc.test.js` も更新）。

## 監視している主な数値（2026年時点）

- 年収の壁：110万（住民税）/ 123万（扶養）/ 130万（社保）/ 150万（特定扶養・勤労学生）/ 160万（本人所得税）
- 住民税：均等割 標準5,000円＋超過課税37府県 / 所得割10% − 調整控除（最大2,500円）
- ⚠️ 基礎控除の上乗せ特例は **2025〜2026年限定**。2027年は要見直し（自動Issueが通知）

## ファイル構成

```
calc.js        … 税計算の純粋ロジック（ブラウザ／Node共用）
app.js         … UI・DOM・保存処理
index.html     … 本体（4タブ）
*.html         … 解説記事・プライバシー
test/          … Node用テスト・健全性チェック
.github/       … 自動保守ワークフロー
```
